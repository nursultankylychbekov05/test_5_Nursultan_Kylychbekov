Описание базы данных warehouse_control

1. ТАБЛИЦЫ И ПОЛЯ (Что где хранится)

В базе данных нет цен и денег, так как по заданию ведется только количественный учет товаров. Всего я решил создать 6 таблиц:

1. warehouses (Склады) — список всех складов компании. 
id — уникальный номер склада (его код).
warehouse_name — название склада (например, «Центральный»).

2. categories (Категории) — группы товаров. 
id — код категории.
category — название группы (например, «Процессоры»). Это обязательное поле.

3. brands (Бренды) — производители товаров.
id — код бренда.
brand — название (например, «Intel»). Это поле может быть пустым (NULL), если у товара нет бренда.

4. products (Товары) — сам список вещей, которые лежат на складах.
id — код товара.
product — точное название товара.
category_id — к какой категории относится товар.
brand_id — какого бренда этот товар (может быть пустым).

5. counterparties (Контрагенты) — это наши партнеры (и те, кто привозит товар, и те, кто его забирает).
id — код партнера.
counterparty_name — название фирмы или имя человека. (например, ООО "Тмыв Денег").

6. transactions (Операции) — главная таблица, куда записывается любое движение товара (приход, расход, перемещение).
id — номер записи.
product_id — какой товар двигался.
qty — сколько штук привезли/увезли.
action_date — дата операции.
source_warehouse_id — склад-отправитель (откуда взяли товар).
target_warehouse_id — склад-получатель (куда привезли товар).
counterparty_id — кто из партнеров участвовал в операции.



2. СВЯЗИ МЕЖДУ ТАБЛИЦАМИ

Все таблицы связаны между собой через ключевые поля (коды ID). Это нужно, чтобы в базе был порядок:
Каждому товару из таблицы products обязательно присваивается категория из таблицы categories и бренд из таблицы brands.
Каждая строчка движения товара в таблице transactions четко знает, какой именно товар (product_id) двигался, на каких складах и с каким контрагентом.



3. КАК ЗАПИСЫВАЮТСЯ ДЕЙСТВИЯ С ТОВАРОМ

Вся логика складского учета происходит в таблице transactions за счет правильного заполнения складов «Откуда» и «Куда»:

А) Поступление товара на склад от контрагента:
Товар пришел извне организации. Склад-отправитель нам неизвестен, поэтому source_warehouse_id оставляем пустым (NULL). В поле target_warehouse_id пишем код нашего склада, куда положили товар. В поле counterparty_id записываем код поставщика, который его привез.

Б) Перемещение со склада на склад:
Товар двигается строго внутри нашей фирмы, внешние люди не участвуют. Поэтому поле counterparty_id оставляем пустым (NULL). В source_warehouse_id пишем склад, с которого товар забрали, а в target_warehouse_id — склад, на который товар привезли.

В) Выдача товара со склада контрагенту:
Товар уходит из организации. В source_warehouse_id пишем наш склад, с которого отгрузили товар. Склад-получатель теперь вне нашей системы, поэтому target_warehouse_id делаем пустым (NULL). В counterparty_id пишем код клиента, который забрал товар.
